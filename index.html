Competitive Analysis

First, we want to research other websites with aggregator functions. I chose 3 websites with a favoriting function. I chose to compare Etsy, Sephora, and Amazon's favoriting functionality and to outline the differences in a comparison chart.

Summary

Etsy's Favorites List: Etsy’s favorite list is the most basic. You can only click into the items from the list and there is no one-click button to add the item to the cart. This leads to less users actually buying the item from the favorites list, because there is no incentive. You can search through items, but, in comparison, an add to cart button would have been more helpful than the search.

Sephora Loves List: Sephora’s “loves” list is more catered for users. Unlike Etsy, they have an easy “Add to Basket” button. This makes it easy for users to buy the item from the liked list. Though the Sephora list doesn’t show the date added, you can filter by recently added. The filter is a trade-off from the search in Etsy and Amazon, where both could be used to find a specific item.

Amazon Shopping List: Amazon’s shopping lists had the most detailed functionality. There are multiple shopping lists you can add, and you can name the shopping lists. This is a good idea, because it allows users to easily navigate to self-categorized liked products (e.i. makeup, fashion, home goods). Also, Amazon has an immediate undo option after you click remove, which is more convenient than a second confirmation click to remove like Etsy, which requires extra work.

Takeaways

For my aggregator website, I want to implement similar features from the websites researched:

  • Sort items by price (high to low, low to high) and by name (A to Z, Z to A) like Sephora
  • Option to remove items from the aggregated list all at once like Amazon

Creating My Own React App

Theme

I decided to create a sandwitch builder app called PB&J Time that allows you to build gourmet peanut butter and jelly sandwitches. The aggregator function would be adding to the cart/adding to the sandwitch. I want the sandwitch to "build" on the screen and include sort/filtering funcitonality.

Visual Design

Initial sketch of web app:

Instead of making a Figma mockup of my web app, I decided to base my webpage design off an existing React app I've worked on: Blueno's Bakery!

Features

  • Sorting feature: can sort by category of item (nut butter, jam, topping) and by flavor (savory, sweet)
  • Filtering Categories: can filter by price (high to low, low to high) and name (A to Z, Z to A)
  • Stackable Sort and Filter: sort and filter can be layered, existing sort applied when filter changes and vice versa
  • Instructions: sort intructions on how to use sandwitch builder app
  • Aggregator Section: cart with items on right, items also added to sandwitch visual
  • Updated count of items in your aggregator: count of items in cart and total price displayed at bottom cart
  • Reset functionality: can reset sort and reset filter with dropdown menu
  • 16 Item Cards Each Displaying: item image, item name, item category, item flavor, item price
  • Button Adds and Removes from Aggregator: button to add to cart from item card, "X" to remove from cart
  • Clear Cart: trash can button to remove all items from cart
  • AI-generated Images: all images enerated with Prodia AI and edited with Photoshop to match asthetic
  • Animation: items "laid" down with animation onto sandwitch visual
